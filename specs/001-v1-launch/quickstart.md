# Quickstart & Validation Guide: JugadaMax V1

**Feature**: [spec.md](./spec.md) | **Plan**: [plan.md](./plan.md) | **Date**: 2026-07-01

This is a run/validation guide to prove V1 works end-to-end once implemented. It contains no
implementation code — see `data-model.md` and `contracts/` for interface details, and `tasks.md`
(generated by `/speckit-tasks`) for the build steps.

## Prerequisites

- Node.js compatible with Next.js 16 and npm.
- Dependencies installed: `npm install`.
- Content records present under `content/` (at least one real casino per vertical, one bonus, one
  author, one review, one guide, one news article).

## Run locally

```bash
npm run dev      # start the dev server
npm run build    # production build (also validates content + SSG)
npm run start    # serve the production build
npm run lint     # eslint
npx tsc --noEmit # type-check the content model and components
```

> Do NOT deploy (Vercel/production) without explicit user confirmation (constitution: Workflow &
> Safety). Do NOT run `npm audit fix --force`.

## Validation scenarios (map to spec Success Criteria)

### 1. Core pages exist and are reachable (SC-002)
- Visit `/`, `/casinos-crypto`, `/casinos-fiat`, `/apuestas`, a `/reviews/[slug]`, `/guias`,
  `/guias/[slug]`, `/noticias`, `/noticias/[slug]`, `/como-evaluamos`, `/divulgacion-afiliados`,
  `/juego-responsable`, `/acerca-de`, `/contacto`.
- **Expected**: each renders; all are reachable from header and/or footer.

### 2. Crypto casino discovery flow (US1, SC-001)
- On a mobile viewport, load `/`.
- **Expected**: hero + quick categories + a prominent Top Crypto Casinos list with rank, rating (if
  present), headline bonus, payment badges, and a CTA — reachable to a casino CTA within ~2 taps.

### 3. Rankings + comparison + methodology (US1/US3, SC-005)
- Open each ranking page.
- **Expected**: entries in `rankByVertical` order; a comparison table usable on mobile; a visible
  link to `/como-evaluamos`.

### 4. Review template (US2, SC-006)
- Open a casino review.
- **Expected**: verdict, editorial rating, balanced pros AND cons, bonus, payments, trust/licensing,
  author byline, and publish/updated date.

### 5. Compliance near CTAs (US4, SC-004)
- On every page containing an affiliate CTA, inspect the CTA and surrounding content.
- **Expected**: affiliate disclosure + 18+ responsible-gambling notice visible without extra
  navigation; outbound link uses `rel="sponsored nofollow"`; a missing `affiliateUrl` yields a
  graceful (non-broken) CTA.

### 6. Guides/news structure (US5)
- Open `/guias` and `/noticias`; open one article of each.
- **Expected**: index lists articles (title + summary); article shows title, body, author byline,
  and date; homepage shows guides + news previews.

### 7. Authenticity audit (SC-007)
- Review all pages/content.
- **Expected**: zero fake winners, fake reviews, fake payout screenshots, fake social proof,
  fabricated urgency/countdowns, or guarantee copy ("dinero fácil", "sin riesgo", "100% gana").

### 8. SEO metadata (SC-008)
- View source / use an SEO inspector on several pages.
- **Expected**: unique title, meta description, canonical, and Open Graph/Twitter tags on every
  page; `<html lang="es-MX">`.

### 9. Structured data — no fake ratings
- Run the pages through a structured-data / rich-results validator.
- **Expected**: `Organization` + `WebSite` site-wide; `Article` on reviews/guides/news with real
  author + dates; **no `AggregateRating`** and **no casino `Review`/`Product` star markup** anywhere.

### 10. Discovery aids (SC-008/FR-023)
- Visit `/sitemap.xml` and `/robots.txt`.
- **Expected**: sitemap lists static pages + all reviews/guides/news; robots allows content and
  references the sitemap.

### 11. Performance & mobile-first (SC-003, FR-018)
- Run Lighthouse (mobile) on `/` and a ranking page.
- **Expected**: primary content readable within ~3s on a typical mobile connection; Performance and
  SEO scores ≥ 90; no layout break at small viewport; no autoplay video / 3D / particles.

### 12. Design fidelity (Principle I, FR-019)
- Visually inspect palette usage.
- **Expected**: navy/graphite backgrounds, gold CTAs, emerald trust badges, off-white text; motion
  is lightweight only and respects `prefers-reduced-motion`.

### 13. Future-readiness (SC-009, FR-025..027)
- Add a new casino/bonus/author/review/article content file and rebuild.
- **Expected**: it appears on the appropriate pages with no code/loader changes; build fails only if
  the record is invalid or references don't resolve.

## Out of scope for V1 (must NOT appear)

- Demo slots / tragamonedas section, playable game embeds.
- User accounts, login, user-generated reviews/ratings.
- Any real-money transaction handling.
